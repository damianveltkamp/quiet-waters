## Screen 1. The Aspiration — Open on what the user wants, not what they're doing wrong. Get an easy first "yes" and establish that the desire is already theirs. (Emotional beat: up / hope.)

The user wants to get closer to god and be more in tune with his/her faith.

## Screen 2. The Problem — Name the exact pain in their own lived experience, so precisely they think "that's literally me." This is validation. (Beat: down / recognition.)

The user spends a lot of time on their phone while they could utilize this to spend more time
reading the lords scripture. On average people spend more then 4 hours per day on their phone.
This time could be used getting closer to god.

## Screen 3. The question - We ask the user how much time they think they spend on their phone

We present them with a few options:

1: 1-3 hours
2: 3-4 hours
3: 4-5 hours
4: 4-5 hours
5: 5-7 hours
6: 7+ hours

## Screen 4. The Stakes — Make the cost of the problem (or size of the opportunity) concrete, usually with a number or vivid framing. Frame as opportunity, never shame, to protect the no-blame tone. (Beat: motivation.)

On this screen we show them how many hours the user will spend this year on their phone.
This number should be calculated based on the answer that was given on screen 3.

## Screen 5. The Promise — State how you'll solve it, in one clear sentence tied to a benefit they care about. Introduce the core mechanic at a high level; tease but don't fully reveal. (Beat: up / relief.)

Our app will present the user with the lords scripture via the creation of custom Iphone backgrounds
with the lords scripture on it. We'll also leverage widgets and or live activity feature of the
Iphone to present the user with the lords scripture.

## Screen 6. The Investment — Have the user put something of themselves in (define their goal, make a choice, enter real data). This personalizes everything downstream and starts the sunk-cost effect early. (Beat: ownership.)

We present the user with a title something like: "This year ill spend these X amount of hours
getting closer to god"
We present them with a "prayer" button that the user should holw to
Above the button we put a text something like "Tap and hold"

## Screen 7. The WOW / Proof — Let them experience the core mechanic end-to-end, personalized with what they just entered — ideally by doing, not watching. This is the payoff that makes the promise believable. (Beat: delight.)

On this screen we let the user create their first custom background, we show them how our product
works.

## Screen 8. Land & Widen — Name what just happened, then zoom out from the single moment to "imagine this every day." Convert the one win into a vision. (Beat: momentum.)

On this screen we show them the background we created and tell the user the background is waiting
for them to be used and to get closer to god.

## Screen 9. The Ask (privileges) — Request the permissions/access the product needs, framed as finishing setup they opted into — riding the investment + WOW so it reads as a step, not an interruption. (Beat: commitment deepens.)

We request the user for access to send notifications so that we can give them the full quiet waters
experience.
